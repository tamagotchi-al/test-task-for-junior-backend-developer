
Сервис для управления задачами с HTTP API на Go с периодичностью

## Что реализовала

Добавила в задачи периодичность. Теперь задача может повторяться.

У задачи появилось поле `rrule` — правило повторения (ежедневно, ежемесячно, по чётным дням и т.д.)
Новый эндпоинт `/api/v1/tasks/period` — показывает задачи за период, повторяющиеся раскладывает на даты
Старые эндпоинты не трогала — обычные задачи работают как раньше

## Как реализовала

**RRule (RFC 5545)** — стандарт Google Calendar
**Вычисление в момент запроса** — копии задач не хранятся в БД, а создаются прямо во время получения ответа
**Валидация** на уровне сервиса

## Скопируйте .env.example в .env и при необходимости измените значения:

cp .env.example .env

## Переменные:

HTTP_ADDR=:8080
DATABASE_DSN=postgres://postgres:postgres@localhost:5432/taskservice?sslmode=disable

## Запуск

docker compose up --build

API: http://localhost:8080/api/v1/tasks
Swagger: http://localhost:8080/swagger/
